js强制类型转换

在javascript中，显式类型转换是这样的；

不过数字也可以隐式的被转换为字符串：

javascript的很多隐式转换经常会让人摸不着头脑，javascript的使用重载，会根据运算符如何被调用来替换运算符的行为。

转为String

javascript使用的二元运算符将两个值连在一起，当javascript尝试将对象转换为字符串时，首先会调用toString()方法，如果toString()不能返回基础类型的值，它会被推给valueOf()函数，如果valueOf()函数也不能返回一个基础类型的值，Javascript会抛出一个TypeError异常。

转为数字

一元运算符的职责是将运算符后的操作数转为数字。这里与字符串的转换过程相反，会先调用valueOf()然后调用toString()

复杂的强转

现在我们回到上方那个复杂的强转来分析

首先：[[]] => [Array[0]], [+[]] => [0]，合起来就是++[[]][0]，本质上就是嵌套的数组被索引0被访问，返回时左边的的一元操作符将其加1，使其变为一个数字。然后这连个值进行合并，由于左边的操作符使数字，右边的是数组，这种组合会被串联，而不是求和：1 + [0] => ‘10’

转为布尔值

使用Boolean()方法可以强制转换任何值为boolean类型，除了以下六个值，其他都自动转为true：

  • undefined
  • null
  • -0
  • +0
  • NaN
  • ‘’

所有对象的布尔值都是true，甚至连false对应的布尔值对象也是true

复杂案例解析

一元运算符可以把空数组转成0，逻辑非运算符把0转成true，等式变成true + true + true，二元运算符号把这些true连着一起，把他们转为数字，就变成了1 + 1 + 1，得到3

js对象的深拷贝

在对对象进行拷贝的时候，普通的“=”操作只是将对象的内存地址拷贝了一份指向新的对象，拷贝后的引用都是指向同一个对象的实例。深拷贝就是在拷贝数据的时候，把数据的所有引用结构都拷贝一份，重新在内存中开辟出一份空间存储拷贝的数据。

JSON.stringfy()和JSON.parse()可以简单有效的搞定简单对象的深拷贝问题。但是这种简单粗暴的方法有其局限性。当值为undefined、function、symbol 会在转换过程中被忽略。。。所以，对象值有这三种的话用这种方法会导致属性丢失。

我们要深拷贝一个数据，必须要遍历他的属性，如果它的对象的属性仍然是对象，那么就继续递归的使用这个方法。

服务器端nodejs实现：

  • 把报头 “Content-Type” 设置为 “text/event-stream”
  • 字段：每个事件之间通过空行来分隔。

字段为空白，表示该行是注释，会在处理时被忽略。
event: 表示该行用来声明事件的类型。浏览器在收到数据时，会产生对应类型的事件。
data: 消息的数据字段。如果该条消息包含多个data字段,则客户端会用换行符把它们连接成一个字符串来作为字段值。
id: 事件ID,会成为当前EventSource对象的内部属性”最后一个事件ID”的属性值。
retry: 一个整数值,指定了重新连接的时间(单位为毫秒),如果该字段值不是整数,则会被忽略.
